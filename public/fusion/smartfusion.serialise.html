Serialisation

Provides an ability to serialise SMART FUSION objects into and from YAML.

SMART FUSION utilises YAML for serialisation. The Declarative Base feature of components is exploited in order to store the data of the components and data types.

This module functions as a plug-in for ruamel.yaml, specified by typ='smartfusion', but for convenience it is recommended to use YAML which defaults with the plug-in enabled.

It is also possible to extend the serialisation for other types with SMART FUSION, via smartfusion.serialise.yaml entry point, typically expected to be used with smartfusion.plugins. The entry point should point to a function which expects a single argument, a ruamel.yaml.YAML instance.

For example:

setup(
    ...
    entry_points={
        'smartfusion.plugins': 'my_plugin = my_package',
        'smartfusion.serialise.yaml': 'my_plugin = my_package:yaml_init_func}
    ...
)
